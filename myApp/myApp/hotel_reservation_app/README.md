# سیستم رزرو هتل (دسکتاپ - Tkinter)

اپلیکیشن دسکتاپ پایتون برای رزرو اتاق هتل به همراه پرداخت درون‌برنامه‌ای **شبیه‌سازی‌شده**
(Mock Payment) برای اهداف تست و آموزش.

## امکانات

- ثبت‌نام و ورود کاربر (رمز عبور به‌صورت هش‌شده ذخیره می‌شود)
- مشاهده و جستجوی اتاق‌های هتل
- رزرو اتاق با انتخاب تاریخ ورود/خروج و محاسبه خودکار مبلغ
- جلوگیری از رزرو دوباره یک اتاق در بازه تاریخی تداخل‌دار
- پرداخت درون‌برنامه‌ای شبیه‌سازی‌شده (اعتبارسنجی شماره کارت/تاریخ انقضا/CVV2)
- تاریخچه رزرو برای هر کاربر + امکان لغو رزرو
- پنل مدیریت اتاق‌ها برای کاربر ادمین (افزودن/ویرایش/غیرفعال‌سازی اتاق)
- پشتیبانی از SQLite (پیش‌فرض) / PostgreSQL / MySQL از طریق SQLAlchemy

## نصب

```bash
cd hotel_reservation_app
pip install -r requirements.txt
```

### اتصال به PostgreSQL

```bash
pip install psycopg2-binary
```

سپس متغیر محیطی را تنظیم کنید (یا مستقیماً در `config.py` تغییر دهید):

```bash
export HOTEL_DB_URL="postgresql+psycopg2://USER:PASSWORD@localhost:5432/hotel_db"
```

### اتصال به MySQL

```bash
pip install pymysql
```

```bash
export HOTEL_DB_URL="mysql+pymysql://USER:PASSWORD@localhost:3306/hotel_db"
```

> نکته: پیش از اتصال باید خود دیتابیس (hotel_db) در سرور PostgreSQL/MySQL از قبل ساخته شده باشد؛
> جداول داخل آن را برنامه (`init_db`) به‌صورت خودکار می‌سازد.

## راه‌اندازی اولیه (ساخت جداول + داده نمونه)

```bash
python seed.py
```

این دستور جداول را می‌سازد، چند اتاق نمونه اضافه می‌کند و یک کاربر ادمین می‌سازد:

```
نام کاربری: admin
رمز عبور:   admin123
```

## اجرای برنامه

```bash
python main.py
```

## ساختار پروژه

```
hotel_reservation_app/
├── main.py              # نقطه شروع برنامه و کنترلر صفحات
├── config.py             # تنظیمات (آدرس دیتابیس و ...)
├── models.py              # مدل‌های SQLAlchemy (User, Room, Reservation, Payment)
├── database.py            # اتصال به دیتابیس و ساخت جداول
├── auth.py                # هش و بررسی رمز عبور
├── payment_service.py     # درگاه پرداخت شبیه‌سازی‌شده (Mock)
├── seed.py                # درج داده نمونه (اتاق‌ها + کاربر ادمین)
├── requirements.txt
└── gui/
    ├── auth_frames.py      # صفحات ورود / ثبت‌نام
    ├── dashboard_frame.py  # صفحه اصلی لیست اتاق‌ها
    ├── booking_frame.py    # انتخاب تاریخ و ثبت رزرو
    ├── payment_frame.py    # پرداخت درون‌برنامه‌ای
    ├── history_frame.py    # تاریخچه رزرو کاربر
    └── admin_frame.py      # پنل مدیریت اتاق‌ها (ادمین)
```

## نکته مهم درباره پرداخت

ماژول `payment_service.py` یک **شبیه‌سازی** است و به هیچ بانک یا درگاه واقعی متصل
نمی‌شود. برای اتصال به یک درگاه واقعی (مثل زرین‌پال، Stripe و ...) کافی‌ست
کلاس `MockPaymentGateway` را با پیاده‌سازی واقعی API آن درگاه جایگزین کنید؛
چون بقیه‌ی برنامه فقط متد `charge()` را فراخوانی می‌کند، نیازی به تغییر
سایر بخش‌های کد نیست.

برای تست تراکنش ناموفق: هر شماره کارتی که با `0000` شروع شود، در شبیه‌ساز رد می‌شود.

## نکات امنیتی برای استفاده واقعی

- رمزهای عبور با نمک تصادفی (salt) و SHA-256 هش می‌شوند؛ برای پروژه‌های واقعی
  استفاده از الگوریتم‌های استانداردتر مثل bcrypt/argon2 توصیه می‌شود.
- در محیط پروداکشن، اطلاعات کارت بانکی هرگز نباید در برنامه یا دیتابیس شما
  ذخیره یا پردازش شود؛ این کار باید کاملاً روی زیرساخت درگاه پرداخت مجاز (PCI-DSS)
  انجام شود.
